Essa dissertação tem como objetivo contribuir com a implementação do ForestWatchers (ver Capítulo \ref{forest_watchers}). A contribuição será realizada através das seguintes etapas: i) desenvolver o módulo de ``\textit{volunteer sensing}'' do projeto; ii) desenvolver a ``interface visual'' do projeto;  iii) integrar esses módulos aos outros já existentes e a plataforma de Ciência Cidadã, PyBossa, e iv) testar e validar todo o sistema com dados PRODES e um conjunto de voluntários.

O módulo de \textit{volunteer sensing} a ser desenvolvido para o projeto ForestWatchers, fornecerá subsídios para melhorar a confiabilidade dos dados. Os voluntários poderão ser solicitados pelo projeto que enviem imagens de áreas pré-determinadas para complementar os dados existentes, ou enviar imagens sem nenhuma solicitação, apenas como caráter informativo, podendo ser utilizado pelo projeto subsequentemente. Assim, as imagens enviadas poderão ser utilizadas para reduzir os falsos-positivos e falsos-negativos, inferidos pelos voluntários. 
%%%%========

Esta proposta tem o objetivo de integrar formalmente os módulos já desenvolvidos (ver Figura \ref{fig:estrutura_atual}). Fornecendo um gerenciamento automático para criação de novas áreas, não tendo a preocupação de compatibilidade, conflitos de transformações dos tipos de dados e estruturas ou tendo que se preocupar com propriedades não funcionais, como confiabilidade, disponibilidade, e segurança \cite{Agt2011}. Um novo módulo também será proposto para coletar dados in-situ, sendo complementar aos módulos já existentes. Característica principal de \textit{Volunteer Sensing}.

A integração permitirá que grandes quantidades de trabalho seja gerenciada com o mínimo de esforço requerido, conforme os usuários comecem adicionar novas áreas a serem monitoradas (ver Figure \ref{fig:automatização}). Outro fato importante que essa integração fornecerá, a distribuição do projeto a novos servidores seria realizado sem complicações, e testes, assegurando-se de que todas as funcionalidades estão como deveriam, será possível.

%%%%========
This proposal has the objective to make the integration of the already developed modules (see Figure \ref{fig:estrutura_atual}) formal. It will provide an automated management for the creation of new areas, becoming seamless for whom may use, not having to concern about compatibility issues, transformations conflicts of data-types and structures or having to concern about non-functional properties such as reliability, avilability and security \cite{Agt2011}. In addition, an in-situ data collection will be proposed to complement the existing architecture of ForestWatchers project. Enabling an alternative way for volunteers to contribute. In-situ data collection is the main characteristic of volunteer sensing, enabling to send data through volunteers' devices. 

%Esta proposta tem como objetivo, formalizar a integração dos módulos já desenvolvidos (ver \ref{fig:estrutura_atual}), para a definitiva automatização do processo de análise de uma nova área de interesse, e adicionar à arquitetura existente a capacidade de coletar dados in-situ (a principal característica de projetos de ciência cidadã do tipo volunteer sensing).

This integration will provide the potential to handle an increasing quantity of work effortless, as users start adding new areas (see Figure \ref{fig:automatização}). Fresh deployments will be easily done, and tests that provide the system assertion. Errors will be treated, considering function and non-functional properties \cite{Agt2011}.

The addition of in-situ data collection to the actual architecture,  will enable the ForestWatchers to have a more consistent final result, with pictures showing real aspects of what volunteers have been contributing. These pictures may add significant improvement to evaluate the results assuring it more quality.

%A automatização do processo tornará o sistema escalável. Assim, a criação de novos subprojetos (outras áreas de interesse) será simples e rápida (ver Fig. \ref{fig:automatização}). Os módulos a serem integrados foram apresentados na Fig. \ref{fig:estrutura_atual}. Adicionando coleta de dados in-situ para a arquitetura atual, almeja-se obter uma maior participação dos usuário, e possibilitar que esses novos dados sejam utilizados pelo módulo que faz as apurações dos resultados, podendo elevar a confiabilidade dos dados.

To integrate a system with already developed modules, the following will be considered:
\begin{enumerate}[i.]
\item Assert that all modules are independently working .  
\item Verify each module's reliability.
\item Perform transformation for between data-types, and structures, as needed.
\item Functional and non-functional properties will be considered \cite{Agt2011}.
\item The construction of a software performance model integration, may be considered \cite{Shin2008}.
\end{enumerate}
%new

To aggregate in-situ capabilities as this proposes, the following will be considered:
\begin{enumerate}[i.]
\item An integrated structure (see Figure \ref{fig:estrutura_modificada}) will be proposed for the ForestWatchers Project. 
	\begin{enumerate}[a)]
		\item In-situ data collection (pictures) will work as an optional input for volunteers.
		\item Both methods must be independent of each other.
	\end{enumerate}
\item To upload the pictures, frameworks of volunteer sensing will be validated.
	\begin{enumerate}[a)]
		\item The framework EpiCollect will be considered (see Section \ref{sec:volunteer_sensing}).
		\item The framework Sensr will be considered (see Section \ref{sec:volunteer_sensing}).
		\item Others framework will be considered as well.
		\item If no other framework had been validated, the development of a new one will be considered.
	\end{enumerate}
\item The uploaded pictures will be stored on the server.
	\begin{enumerate}[a)]
		\item Validation algorithms will verify if the pictures are Geo-tagged, using Exif\footnote{Exchangeable image file format (Exif) - is a widely used file format to aggregate several technical information to audio and picture images \cite{JEITA2002}.} \cite[5]{Catlin-Groves2012}.
		\item If the pictures are Geo-tagged, volunteers will perform similar verification task, of the one required to evaluate non-forest images, to assess its context (see Figure \ref{fig:epicollect_application}).
	\end{enumerate}
\end{enumerate}
